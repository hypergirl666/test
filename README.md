# Digital PMPO

__Digital Проектный практикум__ — это система, которая решает две ключевые проблемы: начинающие Product Manager'ы не знают, с чего начать и как двигаться по воркфлоу, а руководители проектных команд тратят время на ручной сбор отчётов и контроль прогресса.
__Цель проекта__ — объединить обучение через практику и автоматизацию управления командой в единой системе, где бот выступает одновременно как наставник для начинающего Product Manager и как инструмент координации для Project Manager'а.
<br>
## Цель<br>
__Цель__ - создать интеллектуальную систему поддержки в Telegram, которая:
1. Ведёт начинающего Product Manager'а по строгому, но гибкому воркфлоу от идеи до MVP, обучая через практику.
2. Автоматизирует процессы командной работы: регистрация, сбор ежедневных отчётов, формирование сводок, контроль прогресса.
3. Связывает индивидуальное развитие Product Manager'а с общей эффективностью команды.

## Проблемы<br>
1. Начинающие Product Manager'ы сталкиваются с высоким порогом входа: они понимают теорию, но теряются при попытке реализовать продукт с нуля.
2. Им не хватает структурированного воркфлоу, обратной связи и практических подсказок.
3. В то же время Project Manager в студенческих командах вынужден вручную собирать отчёты, напоминать о задачах и отслеживать прогресс, что отвлекает от стратегических задач.
4. Нет интеграции между обучением и реальной работой в команде.

## Назначение
__Digital Проектный практикум__ - это Telegram-бот, который помогает начинающим Product Manager'ам проходить путь от идеи до MVP пошагово, с подсказками, шаблонами и обратной связью. Одновременно бот __автоматизирует__ управление студенческой проектной командой: __регистрирует__ участников, __собирает__ ежедневные отчёты, __формирует__ сводки и __синхронизирует__ прогресс с Miro.

__Система объединяет обучение через практику и командную координацию, выступая одновременно как наставник для PM и инструмент контроля для Project Manager'а__.

## Текущее решение
__DailyBot__ — система управления командой, которая:
1. Позволяет Project Manager создать команду и сгенерировать пригласительную ссылку.
2. Автоматически напоминает каждому участнику (разработчик, тестировщик, DevOps и др.) заполнить ежедневный отчёт с тремя вопросами: Что сделал сегодня? Что планируешь завтра? Есть ли трудности?
3. Формирует сводку по отчётам и отправляет Project Manager'у.
4. Интегрируется с чатом (топик в Telegram) и Miro для визуализации.
5. Позволяет настраивать такие параметры, как время отчета, дни запроса отчета, часовые пояса, напоминалки.
6. Уточняет информацию касаемо текущих задач/блокеров для предоставления этих данных РМ, задавая вопросы от LLM при прохождении дейли опроса.
7. Позволяет участвовать не только в одной проектной команде.
8. Оповещает о добавлении новых фичей в функционал системы.
9. Позволяет кастомизировать вопросы для дейли опросов, если базовые не устраивают.

## Планы на будущее
__Digital Product Manager__ (DPM) — встроенный помощник, который будет __работать совместно__ с __DailyBot__ и который:
1. Проводит онбординг.
2. Помогает сформулировать гипотезу, провести исследование, построить бэклог, приоритизировать по RICE, создать roadmap.
3. Даёт обратную связь, проверяет прогресс, синхронизирует результаты с Miro.
4. Доступен только для участников с ролью "Product Manager".

## Контекст взаимодействия
На основе __BPMN__ схем происходит следующее:
1. Процесс инициирует Заказчик, описывающий цель проекта.
2. Куратор уточняет детали.
3. Project Manager выбирает методологию, формирует требования, создаёт команду в боте.
4. Product Manager работает с Digital PMPO, проходит воркфлоу, ставит задачи разработчикам.
5. Разработчик получает задачу, выполняет её, проходит код-ревью (по напоминанию бота), отправляет на тестирование.
6. После тестирования и, при необходимости, доработки — задача идёт на релиз через DevOps.
7. Каждый день бот собирает отчёты, формирует сводки и уведомляет PM о проблемах.
8. В конце спринта — автоматический финальный отчёт.
