# Часовые пояса: изменения и поведение

Этот документ описывает все изменения, связанные с учётом часовых поясов в боте, а также текущее поведение планировщика и пользовательских сообщений.

## Что изменилось

- Приглашения на дейли теперь показывают дедлайн с явным указанием часового пояса команды.
  - Формат: `Дедлайн: сегодня|завтра, HH:MM (МСК|ЕКБ)`.
- Расчёт дедлайна для приглашений выполняется в часовом поясе команды (локальная «now» для корректной дельты и меток «сегодня/завтра»).
- Планировщик задач (утро/вечер/отчёт) создаёт Cron-триггеры с таймзоной команды, что корректно работает для мультикоманд.
- Документация обновлена: добавлен раздел «Часовые пояса» в `bot/README.md`.

## Откуда берётся часовой пояс

- Поле `team["timezone"]` (например, `Europe/Moscow`, `Asia/Yekaterinburg`).
- Если у команды не задано — используется глобальная `TIMEZONE` из `.env`.
- Сокращения в тексте сообщений:
  - `Europe/Moscow` → «МСК»
  - `Asia/Yekaterinburg` → «ЕКБ» (по умолчанию)

## Затронутые места кода

- `bot/handlers/daily_handlers.py`
  - Функция `_send_daily_invitation(...)`:
    - Расчёт дедлайна исходит из расписания команды и её таймзоны (через APScheduler `CronTrigger(..., timezone=team_timezone)`).
    - «Текущее время» для расчёта дельты берётся в таймзоне команды.
    - В строку дедлайна добавлен ярлык таймзоны: `({timezone_str})`.
- `bot/utils/scheduler_jobs.py`
  - При настройке задач команды (`setup_team_scheduler_jobs`) `CronTrigger` создаётся с `timezone=team_timezone`.

## Пользовательские эффекты

- Сотрудники видят дедлайн с указанием часового пояса, что исключает путаницу при распределённых командах.
- В мультикомандной конфигурации рассылки для разных команд идут согласно их локальному времени.

## Обратная совместимость

- Если поле `team["timezone"]` не задано, берётся `TIMEZONE` из окружения.
- При отсутствии корректной таймзоны используется дефолт из конфига, чтобы не ломать поведение.

## Пример приглашения

```
☀️Доброе утро☀️
Пришло время дейли!

У вас есть 1 ч. 45 мин. на отправку отчёта
Дедлайн: сегодня, 10:00 (ЕКБ)

Можно отвечать текстовыми и голосовыми сообщениями🗣
Готовы ответить на несколько вопросов?
```

## Проверка и тестирование

- Изменённые файлы проходят линт.
- Для проверки вручную:
  1) Задать у команды `timezone=Europe/Moscow` и отправить утренние приглашения — проверить `(МСК)` и корректность «сегодня/завтра».
  2) Задать `timezone=Asia/Yekaterinburg` — проверить `(ЕКБ)` и корректность времени.
  3) Убедиться, что расписание запускается в ожидаемое локальное время для каждой команды.
