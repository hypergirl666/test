# Настройки времени команды — актуальная реализация

## Описание

Новый функционал позволяет менеджерам команд настраивать индивидуальное время проведения дейли и отправки отчетов для своих команд.

## Возможности

### Для менеджеров:
- ⏰ Настройка времени утреннего опроса (по умолчанию: 09:00)
- 🌆 Настройка времени вечернего опроса (по умолчанию: 22:00)
- 📊 Настройка времени отправки отчетов (по умолчанию: 10:00)
- 📅 Настройка дней недели для каждого типа времени
- ✅ Валидация логических ограничений времени

### Логические ограничения:
1. **Отчёты** позже утреннего опроса
2. **Вечерний опрос** позже отчётов
3. Формат времени: `HH:MM` (например: `09:30`, `14:15`)

## Структура базы данных

### Поля в таблице `teams`:

| Поле | Тип | По умолчанию | Описание |
|------|-----|--------------|----------|
| `morning_time` | TEXT | `'09:00'` | Время утреннего опроса |
| `evening_time` | TEXT | `'22:00'` | Время вечернего опроса |
| `report_time` | TEXT | `'10:00'` | Время отправки отчетов |
| `report_days` | TEXT | `'tue,wed,thu,fri'` | Дни отправки отчётов |

**Важно:** Дни `morning_days` и `evening_days` не хранятся — вычисляются динамически из `report_days` (`bot/utils/day_utils.py`).


## Использование

### Для менеджера:

1. **Открыть меню менеджера** - нажать кнопку "⏰ Время"
2. **Выбрать тип времени** для настройки:
   - 🌅 Утренний опрос
   - 🌆 Вечерний опрос
   - 📊 Время отчетов
   - 📅 Дни недели
3. **Выбрать время** из предустановленных вариантов
4. **Выбрать дни недели** — интерактивная клавиатура с `✅/❌` для `report_days`; изменения применяются сразу
5. Сохранение происходит автоматически при каждом изменении

### Доступные пресеты времени:
```
08:00, 08:30, 09:00, 09:30, 10:00, 10:30,
11:00, 11:30, 12:00, 12:30, 13:00, 13:30,
14:00, 14:30, 15:00, 15:30, 16:00, 16:30,
17:00, 17:30, 18:00, 18:30, 19:00, 19:30,
20:00, 20:30, 21:00, 21:30, 22:00, 22:30,
23:00, 23:30
```


## Техническая реализация

### Новые файлы:
- `handlers/team_time_handlers.py` - обработчики настроек времени

### Обновленные файлы:
- `config.py` - добавлены константы и функции валидации
- `core/database.py` - новые функции для работы с настройками времени
- `core/states.py` - новые состояния FSM
- `utils/keyboards.py` - новые клавиатуры
- `utils/text_constants.py` - новые текстовые константы
- `utils/scheduler_jobs.py` - обновлен планировщик для работы с командами
- `utils/day_utils.py` - функции для работы с днями недели

### Функции базы данных:
- `db_get_team_time_settings(team_id)` - получение настроек времени команды
- `db_update_team_time_settings(...)` - обновление настроек времени
- `db_get_teams_by_time(time_type, time_value)` - поиск команд по времени
- `db_get_all_teams_with_times()` - получение всех команд с настройками времени

### Планировщик:
- `setup_team_scheduler_jobs(scheduler, team_id)` — настройка задач (утро/вечер/отчёт) для команды
- `update_team_scheduler_jobs(scheduler, team_id)` — обновление задач команды
- `remove_team_scheduler_jobs(scheduler, team_id)` — удаление задач команды
- `setup_all_teams_scheduler_jobs(scheduler)` — настройка задач для всех команд
- `update_team_scheduler(team_id)` — глобальный вызов обновления через `SchedulerManager`

### Дни недели:
- `get_computed_team_days(report_days)` — возвращает `morning_days` и `evening_days` из `report_days`
- Интерактивное переключение `report_days` в `team_time_handlers.py` → пересчёт `morning_days`/`evening_days` и мгновенное сохранение

## Обратная совместимость

- ✅ Существующие команды автоматически получают настройки по умолчанию
– Формат дней: используем строки с запятыми (`mon,tue,...`). Старые дефисы не применяются.
– Все изменения применяются без перезапуска.

## Логирование

Все операции с настройками времени логируются:
- Изменение настроек времени
- Создание/удаление/обновление задач планировщика
- Ошибки валидации и форматирования времени

## Безопасность

- Валидация всех вводимых значений
- Проверка прав доступа (только менеджер команды)
- Защита от SQL-инъекций через параметризованные запросы
